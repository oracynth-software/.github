# Oracynth

A family archive platform with an evidence-first data model and inclusive architecture.

Oracynth helps families organize photos, documents, oral histories, and records into a unified, portable archive. Every fact traces back through citations to sources. Any two people can be connected through researcher-defined relationships, with no assumptions about gender, quantity, or biology.

The product is a browser-based PWA with client-side SQLite and git-backed storage. Your archive lives in a standard git repository using [GLX (GENEALOGIX)](https://github.com/genealogix/glx), an open YAML specification for genealogical data. You own the files. You can clone them. Nothing is locked in.

### Principles

- **Oral histories are evidence.** Audio recordings and transcripts carry the same weight as documents, vital records, or DNA tests.
- **Connections over lineage.** The graph supports chosen family, adoption, godparents, and any relationship a family needs.
- **Assertions over conclusions.** Conflicting evidence is preserved. Git commit history provides the audit trail.
- **Data ownership is structural.** Open format, standard git, no export tax.
